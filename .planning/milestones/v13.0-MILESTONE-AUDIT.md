## Tech Debt

### Phase 129 — QUAL-02 Scope Limitation

**Non-blocking. QUAL-02 is partially satisfied:**
- Handler functions whose bodies contain `Request.*` accessor calls do infer the `Request` type correctly — the test `e2e_http_middleware_inferred` confirms this with `fn handler(request) do` (no annotation).
- Passthrough middleware whose body only contains `next(request)` still requires `:: Request` annotation. Removing it causes SIGBUS under live HTTP load (type variable generalizes to `forall T`; codegen emits LLVM `{}` type).
- The fixture `tests/e2e/stdlib_http_middleware_inferred.mpl` has a misleading comment at lines 3-4 claiming both patterns work without annotation. This is incorrect per SUMMARY 129-02.
- The original production fixture `tests/e2e/stdlib_http_middleware.mpl` correctly retains `:: Request` annotations and should be treated as the authoritative example.

**Recommended cleanup:** Update the comment in `tests/e2e/stdlib_http_middleware_inferred.mpl` to clarify the scope of QUAL-02.

### Requirements.md Checkbox Staleness

**Non-blocking.** QUAL-01 and QUAL-02 in REQUIREMENTS.md show `[ ]` (unchecked) and "Pending" in the traceability table. Both were addressed in Phase 129. QUAL-01 is fully satisfied; QUAL-02 is partially satisfied with documented scope.

**Recommended cleanup:** Update REQUIREMENTS.md to mark QUAL-01 `[x]` and QUAL-02 with a scope note.

### Phase 131 SUMMARY Minor Typo
