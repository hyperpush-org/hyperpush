Add this arm alongside the existing `Some('$') if peek_next == Some('{')` arm:

```rust
Some('#') if self.cursor.peek_next() == Some('{') => {
    let content_end = self.cursor.pos();
    self.cursor.advance(); // consume '#'
    self.cursor.advance(); // consume '{'
    let interp_end = self.cursor.pos();
    self.state_stack.push(LexerState::InInterpolation { brace_depth: 0 });
    self.pending.push(Token::new(TokenKind::InterpolationStart, content_end, interp_end));
    if content_end > start {
        return Token::new(TokenKind::StringContent, start, content_end);
    } else {
        return self.pending.remove(0);
    }
}
```

Also update the escape handler to recognize `\#{` as an escape sequence for a literal `#{`. The existing `Some('\\')` arm already consumes two characters (the `\` plus the next char). This handles `\#` correctly since it will consume `\` then `#`, yielding `#` in content. No extra change needed — `\#{` will produce `#{` in output because the backslash consumes the `#`, leaving `{` as the next content character. Verify this reasoning: yes, `\#` → the backslash arm consumes `\` then `#`, leaving `{` as normal content (not triggering `#` + `{` check since we already consumed the `#`). This is correct.

Note: Do NOT remove or modify the existing `Some('$') if peek_next == Some('{')` arm — keep it for backward compatibility. Both `${` and `#{` emit identical tokens.
