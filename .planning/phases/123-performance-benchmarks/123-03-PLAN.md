  <action>
Create `benchmarks/run_benchmarks.sh` — a shell script that:

1. Builds and starts all four benchmark servers in background
2. Waits for each server to be ready (curl health check loop)
3. For each server x each endpoint (/text, /json), runs:
   - 10s warmup: `wrk -t$(nproc) -c100 -d10s http://localhost:PORT/ENDPOINT`
   - 3 timed runs: `wrk -t$(nproc) -c100 -d30s --latency http://localhost:PORT/ENDPOINT`
4. Captures and parses wrk output (Requests/sec, Latency 50th, 99th percentile)
5. Averages the 3 runs for each measurement
6. Prints a formatted summary table to stdout
7. Kills all server background processes on exit (trap EXIT)

Port assignments:
- Mesh: 3000 (run with `meshc run benchmarks/mesh/bench.mpl` or the compiled binary)
- Go: 3001 (run with `go run .` in benchmarks/go/)
- Rust: 3002 (run with `cargo run --release` in benchmarks/rust/)
- Elixir: 3003 (run with `mix run --no-halt` in benchmarks/elixir/ after `mix deps.get`)

Server startup: wait up to 30s for each server with:
```bash
wait_for_server() {
  local port=$1
  local max_wait=30
  local elapsed=0
  until curl -sf "http://localhost:$port/text" > /dev/null 2>&1; do
    sleep 1
    elapsed=$((elapsed + 1))
    if [ $elapsed -ge $max_wait ]; then
      echo "Server on port $port failed to start" >&2
      exit 1
    fi
  done
}
```

wrk output parsing: wrk outputs lines like:
- `Requests/sec:  12345.67`
- `Latency    50.00%    1.23ms`  (when --latency flag used)
- `99.000%    5.67ms`

Parse with awk/grep. Average 3 runs for req/s, p50, p99.

Output format (print to stdout at end):
```
== Benchmark Results (100 connections, 30s x3 averaged) ==

Endpoint: /text (text/plain)
Language   Req/s       p50       p99
------------------------------------
Mesh       XXXXX       Xms       Xms

Elixir     XXXXX       Xms       Xms

Endpoint: /json (application/json)
Language   Req/s       p50       p99
------------------------------------
...

Full results: benchmarks/RESULTS.md
```

For Mesh server startup: attempt `meshc run benchmarks/mesh/bench.mpl` first; if meshc is not in PATH, print a warning and skip Mesh but continue with Go/Rust/Elixir. The script must be robust — partial results are acceptable.

Make the script executable: `chmod +x benchmarks/run_benchmarks.sh`.

```python
import matplotlib.pyplot as plt
import numpy as np

languages = ['Mesh', 'Go', 'Rust', 'Elixir']
text_rps = [MESH_TEXT_RPS, GO_TEXT_RPS, RUST_TEXT_RPS, ELIXIR_TEXT_RPS]
json_rps = [MESH_JSON_RPS, GO_JSON_RPS, RUST_JSON_RPS, ELIXIR_JSON_RPS]

x = np.arange(len(languages))
width = 0.35

fig, ax = plt.subplots(figsize=(10, 6))
ax.bar(x - width/2, text_rps, width, label='/text', color='#4C72B0')
ax.bar(x + width/2, json_rps, width, label='/json', color='#DD8452')

ax.set_xlabel('Language')
ax.set_ylabel('Requests per second')
ax.set_title('HTTP Throughput: Mesh vs Go vs Rust vs Elixir\n(100 connections, localhost, 30s averaged)')
ax.set_xticks(x)
ax.set_xticklabels(languages)
ax.legend()
ax.yaxis.set_major_formatter(plt.FuncFormatter(lambda x, _: f'{x:,.0f}'))

fig.tight_layout()
plt.savefig('benchmarks/throughput.png', dpi=150)
```
