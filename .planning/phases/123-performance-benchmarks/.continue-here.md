## Fix Elixir version in Dockerfile.servers

**Error:** `You're trying to run :bench on Elixir v1.12.2 but it has declared in its mix.exs file it supports only Elixir ~> 1.14`

**Root cause:** Runtime stage installs `elixir` via `apt-get` from Ubuntu 22.04 repos = Elixir 1.12.2.

**Fix:** Install Elixir 1.14+ from a different source. Options:
A. Install from official Erlang Solutions repo (adds erlang-solutions.com apt source, installs `elixir=1.14.*`)
B. Use `asdf` to install specific Elixir version
C. Download Elixir prebuilt release from GitHub (simplest, no extra repo)

**Recommended (option C):** Download Elixir 1.17 prebuilt binary from GitHub releases:
```dockerfile
# Remove erlang-dev elixir from apt-get line
# Add after Erlang install:
RUN wget -q https://github.com/elixir-lang/elixir/releases/download/v1.17.3/elixir-otp-26.zip \
    -O /tmp/elixir.zip \
    && unzip -q /tmp/elixir.zip -d /usr/local/elixir \
    && rm /tmp/elixir.zip
ENV PATH="/usr/local/elixir/bin:${PATH}"
```
Note: Elixir prebuilt zip needs compatible OTP/Erlang. Ubuntu 22.04 erlang-dev is OTP 24. Elixir 1.17 needs OTP 24+. Check compatibility.

Actually safest: use erlang-solutions.com repo which has both compatible Erlang + Elixir:
```dockerfile
RUN wget -q https://packages.erlang-solutions.com/erlang-solutions_2.0_all.deb \
    && dpkg -i erlang-solutions_2.0_all.deb \
    && apt-get update \
    && apt-get install -y esl-erlang elixir \
    && rm erlang-solutions_2.0_all.deb \
    && rm -rf /var/lib/apt/lists/*
```
This replaces the `erlang-dev elixir` in the current apt-get line.
