<decisions_made>

- Fix 1 (parser): `parse_block_body` already emits a BLOCK node which is already `Expr::Block` in the AST; `infer_case` already handles it via `infer_expr` dispatch. No typechecker changes needed.
- Fix 2 (mutual recursion): Simple pre-registration with fresh type vars + unify-in-infer_fn_def. Works for monomorphic mutual recursion; polymorphic still has limitations (acceptable).
- Fix 3 (arity mangling separator): `__` + arity number (e.g. `slugify__1`, `slugify__2`). Valid in all backends.
- Fix 3: Only mangle when there are MULTIPLE pub fns with the same name. Single-arity functions keep plain names for backward compat.
- Fix 3: `overloaded_call_targets: FxHashMap<TextRange, String>` in TypeckResult to pass resolved mangled names from typechecker to MIR lowerer.
- Fix 3: For qualified module calls (Slug.slugify), intercept in infer_call by checking qualified_modules for name__N before falling through to normal FieldAccess inference.
- Fix 3: Pub overloaded fns use unqualified mangled names (slugify__1) since pub fns don't get module-prefixed per qualify_name logic.
